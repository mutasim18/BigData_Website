Description of Tools Used

  1. Google Big Query:

    Big Query is one of the major services from Google’s cloud platform. It is used to analyze large quantity of data. It is a platform as service (PaS) which uses queries just like we use in SQL. It has an interactive ad hoc query system which is also able to use functions of machine learning models. Another thing to mention is its compatibility with Google’s cloud ecosystem. There are two lairs in big query. One store and preprocess the data. Another one analyzes and gives output. These two layers work dynamically which increases the performance of the system.

  2. Azure Machine Learning Studio:

    Machine Learning Studio is a service that is offered by Azure platform. As the name suggests, it is used for machine learning purposes. But the unique thing about this service is that It uses a drag-and-drop of nodes. These nodes or should I say modules are part of the machine learning process. So technically, if you know the process behind building a model, you can pick the nodes you want, connect them sequentially like a diagram and let do the studio the back-end stuff. No knowledge of coding is needed in this process. But to get better performance we can customize the nodes within the studio or just use python.

  3. Jupyter Notebook:

    Jupyter Notebook is a open source computational environment that we can interact on the cloud, hosted on jypyterHub. It was built using open source libraries and mainly used for python applications. Most convenient part of using Jupyter is that we can use it on any browser. In the notebook it has cell like structures which is used for input and output of a code.

  4. Amazon S3:

    Within Amazon Web service ecosystem, they offer Amazon Simple Storage Service(S3). Just like we use google drive, Amazon S3 is used for cloud storage. But it is quite different. The data in S3 is organized into sets of buckets. These are assigned by the user from the console. Each of these buckets have their own IDs. Thus it can be only accessed by someone who has the ID.

  5. GCS:

    Google Cloud Storage (GCS) is a cloud storage service that is offered by Google Cloud Platform (GCP). This is not the same as google drive. GCS is part of the google ecosystem where we can store data and can only access through GCP. Unlike Amazon S3, GCS is a infrastructure as a service (iaas).

  6. SageMaker:

    SageMaker is a part of Amazon web service(AWS). It is a cloud-based platform which is used for machine learning purposes. Best practices are deploying it on embedded applications. It comes with pre-built in machine learning models. But if needed, anyone can make their own version of model from scratch. Best part of using Sagemaker is the integration with AWS. It is managed by web API’s using popular programming languages.

  7. Vertex AI:

    Vertex AI is part of google cloud platform. It is a tool that combines data engineering, ML engineering workflows. Major benefit of using this service is that you don’t really need to know coding. If you know the workflow then just select the instruction you need and vertex AI will manage it for you in the background. Strong implication of using this service is the integration of GCP.

  8. Docker:

    Docker is a tool that is used for deploying applications with Virtualization. It uses the docker engine which creates containers. The applications are virtualized in these containers, which are safe and lightweight to use. For specific applications this is a necessary step for development.
