Method Used

The study explores the application of various machine learning models on a housing prices dataset obtained from Kaggle. The dataset contains 546 rows and 13 columns, including features like price, area, bedrooms, bathrooms, etc. The models are implemented and compared on different platforms including Google BigQuery, Azure Machine Learning Studio, and JupyterNotebook. In Google BigQuery, Linear Regression and XGBoost models are applied, but LightGBM is not available. In Azure Machine Learning Studio, Linear Regression is implemented, but XGBoost is not directly available. For LightGBM, a similar approach to XGBoost is used with some modifications in preprocessing.

In AWS SageMaker, attempts are made to deploy the XGBoost regression model, but the process encounters difficulties and ultimately fails. The steps involve preparing model files, uploading to Amazon S3, building a Docker image, creating a SageMaker model, configuring an endpoint, and deploying it. Despite efforts, the deployment is unsuccessful. Following the failure on AWS SageMaker, the study attempts to utilize Google Cloud Platform's Vertex AI for deployment but faces similar challenges in deploying the model to the endpoint. The steps involved in preparing the model files, creating a Docker image, and running a Flask app are mentioned, but the deployment process remains unsuccessful. Overall, the study presents efforts to implement machine learning models on different platforms for housing price prediction, encountering challenges primarily in the deployment phase on both AWS SageMaker and GCP Vertex AI.
